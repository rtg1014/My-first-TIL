2. 1주차 주요 개념 키워드 적어보기

- 버전 관리 : 깃을 통한 업무 효율을 위한 높이기 위한 일종의 툴? 이 되겟다. 내 자료에 커밋을 달고sourcetree 를 통한 로컬 리포와 멀티 리포? 를 연결할 수 있고,
- git 초기화 (initialize, init) : 내가 지정한 폴더를 git 폴더로 바꾼는 작업을 말함
- add / staging : git 초기화하여 소스트리에 지정된 내 폴더를 커밋등의 작업을 위해 올리고 내리고 하는 작업? 이 되겟다 
- commit (커밋) : 내가 수정한 자료들이 누가 왜 어디를 바꿧는지 알려주는 깃의 내용들을 등록? 해주는 작업
- commit 내역(history) :  커밋한 내용들과 커밋한 과거 기록들이 나온다.
- push : 로컬에서 커밋이 끝난 작업물을 멀티리포로 올리는 작업 
- pull : 멀티리포에서 내려받는 작업
- clone : 멀티는 있는데 로컬이 없을때 사용하는 방식 
- tracking : 쫒아간다는 의미로 로컬리포에서 멀티 리포로 올릴려는 작업 그 자체를 의미한다?
